# Crisis CMS — マルチテナント版 独自CMS（共同編集 + AI連携）

ドラッグ&ドロップでページを作れる独自CMSのフルスタック版です。
**マスター（運営者）が複数のサイトとログインアカウントを発行し、各エンドユーザーは自分のサイトだけをアイパスでログインして編集・公開できる** マルチテナント構成です。

## 2種類のログイン

| 役割 | できること | ログイン後の画面 |
|------|-----------|----------------|
| **マスター（運営者）** | 全サイトの作成・削除、エンドユーザーのアカウント発行・パスワード再発行、各サイトの編集に入る | マスター管理画面（サイト一覧） |
| **エンドユーザー** | 割り当てられた自分のサイトだけを編集、公開/プレビュー、アカウント設定 | ビルダー画面（自サイト固定） |

エンドユーザーの権限は3段階: **オーナー（全権）／編集者／閲覧のみ**。アカウントはすべてマスターが発行します（自己登録なし）。
ログインIDはメールアドレスです。同じログイン画面で、役割に応じて自動的に適切な画面へ振り分けられます。

## 主な機能

- **ドラッグ&ドロップ編集** — 13種のブロック（ヒーロー / 見出し / テキスト / 画像 / ボタン / リスト / 引用 / 区切り / 余白 / 動画 / カラム / HTML）
- **リアルタイム共同編集** — WebSocketで複数人の編集が即座に同期。上部に同時編集中メンバーのアイコンを表示
- **ユーザー / 権限管理** — 管理者・編集者・閲覧者の3ロール。管理者はメンバーの追加・権限変更・削除が可能
- **サーバーDB保存** — SQLiteに自動保存（編集の都度）。直近20件のリビジョンも自動保存
- **✨ Claudeデザイン提案** — 「カフェの紹介ページを作って」などの指示でページ構成を自動生成。末尾追加／全置換を選択可能。デザイン助言だけもらうことも可能
- **✨ AI画像生成** — 画像ブロックでプロンプトを入力すると画像を生成して自動挿入（OpenAI gpt-image-1）
- **デザイン設定** — フォント / 配色 / 幅 / 背景など
- **静的サイト書き出し** — そのまま公開できるHTMLを出力
- **🔗 参考URL取込** — 参考サイトのURLを貼ると、構成・配色を分析して近いレイアウトのブロックを自動生成（複製ではなく構成の再現案。文章は仮テキストに置換されるので自分用に調整）
- **📋 テンプレート集** — コーポレート／飲食店／ポートフォリオ／LP の完成済みレイアウトから選んで開始
- **✉ 問い合わせフォーム** — フォームブロックを設置（項目は自由に追加・編集）。公開ページから送信された内容はDBに保存され、上部「📨 問い合わせ」で一覧確認
- **🖼 メディアライブラリ** — アップロード画像・AI生成画像をサイトごとに一覧管理し、画像ブロックから再利用
- **🔍 SEO設定** — ページごとにタイトル・ディスクリプション・OGP画像を設定し、公開HTMLの`<title>`/`<meta>`/OGPに反映
- **📊 アクセス解析ダッシュボード** — 専用のフルスクリーン画面。期間指定・日別/週別/月別の切替・ページ絞り込みに対応し、KPIカード（PV・ユーザー・新規ユーザー・セッション近似・PV/セッション・参照元数）、PV/ユーザーのエリアグラフ（Chart.js・ローカル同梱でオフライン動作）、流入元別URL・ページ別アクセスのテーブルを表示。訪問者IDで新規/リピートを判定
- **✨ 解析AI** — 蓄積したアクセスデータをClaudeが分析し、傾向と具体的な改善アクションを提案
- **📝 AIブログ生成＆コラム投稿** — テーマとトーンを指定するとClaudeが記事（タイトル・抜粋・本文HTML）を生成。下書き／公開／**予約投稿**（指定日時に自動公開）を管理し、公開記事は `/s/スラッグ/blog` のブログ一覧と各記事ページとして自動配信（サイトのナビにもブログリンクを追加）

### さらに先進機能
- **本格アクセス計測** — 滞在時間・スクロール深度・**コンバージョン計測**（フォーム送信／ボタンクリックを目標化）・デバイス判定まで実測。ダッシュボードはコンバージョン数/率・PV・セッション・ユーザー・新規・エンゲージ率・直帰率・平均滞在時間・PV/セッション・参照元・デバイス別を表示
- **A/Bテスト** — ヒーローのタイトルを2案で50/50に出し分け、コンバージョンに variant ラベルを付与して比較
- **編集履歴・ワンクリック復元** — ページ保存ごとに自動で版を記録（直近20件）。任意の版へ即時ロールバック
- **SEO自動化** — `sitemap.xml` / `rss.xml` / `robots.txt` の自動生成、全ページ/記事に **JSON-LD 構造化データ** を出力、**AIによるSEOタイトル・ディスクリプション自動提案**
- **問い合わせ通知** — フォーム送信を Webhook（Slack等）へ通知、SMTP設定時はメール通知（`SMTP_URL`）
- **端末プレビュー切替** — 編集中にPC／タブレット／スマホ表示を即確認

### 最先端（AI自律・運用）機能
- **💬 会話型ビルダー** — 「トップにお客様の声を追加して、最後に予約フォームも」のように自然文で指示すると、現在のページのブロックをAIが編集・反映
- **🚀 AIグロースエージェント** — 週次で自動（手動実行も可）にアクセス解析とページ内容を分析し、コンバージョン改善の施策を生成して「承認キュー」に提案。オーナーは対応済み／却下で管理
- **🧪 A/B自動勝者判定** — ヒーローのタイトル2案の表示数・コンバージョン率を集計し、勝者をワンクリックで本採用
- **♿ AIアクセシビリティ／表記チェック** — ページ本文の代替テキスト・見出し構造・誤字脱字・表記ゆれをAIが点検
- **🗂 操作ログ（監査）** — 公開・復元・A/B採用などの操作を記録して一覧
- **💾 バックアップ取得／復元** — サイト（ページ・記事・設定）をJSONで書き出し、ワンクリックで復元
- **📊 マスター使用量・プラン管理** — サイトごとの今月PV・PV上限・プラン（free/pro/business）を管理（SaaS運用向け）
- **⚡ パフォーマンス最適化** — 画像・動画の遅延読み込み（`loading=lazy` / `decoding=async`）でCore Web Vitalsに配慮

## セットアップ

必要環境: **Node.js 22.5 以上**（組み込みの `node:sqlite` を使うためネイティブビルド不要。`npm install` だけで動きます）

```bash
cd mycms-server
npm install
cp .env.example .env      # Windowsは copy .env.example .env
# .env を編集（下記参照）
npm start
```

ブラウザで http://localhost:3000 を開く。初回起動時、`.env` の `MASTER_EMAIL` / `MASTER_PASSWORD` でマスターアカウントが自動作成されます（既定 `master@example.com` / `master123`）。

### .env の設定

| 変数 | 説明 |
|------|------|
| `PORT` | 待ち受けポート（既定 3000） |
| `DB_PATH` | DBファイルの保存先（既定 `mycms.db`） |
| `ANTHROPIC_API_KEY` | Claude（デザイン提案）用。https://console.anthropic.com/ |
| `ANTHROPIC_MODEL` | 既定 `claude-sonnet-4-6` |
| `OPENAI_API_KEY` | 画像生成用。https://platform.openai.com/ |
| `OPENAI_IMAGE_MODEL` | 既定 `gpt-image-1` |
| `MASTER_EMAIL` / `MASTER_PASSWORD` / `MASTER_NAME` | 初回マスター（運営者）アカウント |

> APIキーが未設定でもCMS本体は動作します。AI機能を使ったときだけ「キー未設定」のエラーが表示されます。

## 使い方

### マスター（運営者）
1. マスターアカウントでログイン → マスター管理画面が開く
2. 「＋ 新しいサイトを作成」でサイト名・スラッグ（公開URL `/s/●●●`）と、必要ならオーナーのログインアカウントを発行
3. 既存サイトの「アカウント発行」で、そのサイト専用の編集者アカウントを追加（権限を選択）
4. 「編集に入る →」で任意のサイトのビルダーを開いて直接編集も可能（左上「← 管理」で戻る）
5. 各アカウントの「PW再発行」「削除」も管理画面から

### エンドユーザー
1. マスターから渡されたメール（ID）とパスワードでログイン → 自分のサイトのビルダーが開く
2. 左の「ページ」で新規ページ作成、ブロックをドラッグして配置（見出し・本文はキャンバスで直接編集）
3. 右の「✨ AIデザイン提案」に要望を入力 →「構成を生成」。画像ブロックでは「AIで画像生成」
4. 上部「公開する」で公開ON/OFF。「🔗 表示」で公開ページ（`/s/スラッグ/`）を別タブ表示
5. 「アカウント」から表示名・パスワードを変更
6. 別の人が同じページを開くと編集がリアルタイム同期（上部にメンバーのアイコン）

## 本番運用メモ

- HTTPSの背後（リバースプロキシ等）に置くと、WebSocketは自動的に `wss://` を使います
- データは `mycms.db`（SQLite）に保存されます。バックアップはこのファイルをコピーするだけ
- 安価なVPSや社内サーバーで `npm start`（必要なら pm2 / systemd で常駐化）
